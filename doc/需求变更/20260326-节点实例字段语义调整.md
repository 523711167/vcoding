# 节点实例字段语义调整

## 变更原因

`tb_workflow_node_instance` 当前使用：

- `node_id`
- `node_name`
- `node_type`

这三个字段保存运行时节点实例对应的“节点定义快照”。  
但现有注释和代码命名容易让人误解为“节点实例自身字段”，语义不够准确。

因此本次统一收敛为“节点定义”口径，避免后续运行层、审批记录、节点查询继续混淆“定义节点”和“节点实例”两个概念。

## 变更内容

### 1. 数据库字段名与语义调整

`tb_workflow_node_instance` 三个字段统一调整为：

- `definition_node_id`：节点定义 ID
- `definition_node_name`：节点定义名称（冗余）
- `definition_node_type`：节点定义类型（冗余）

本次同时修改数据库字段名和语义表达。

### 2. Java 实体命名调整

`WorkflowNodeInstance` 实体统一改为：

- `definitionNodeId`
- `definitionNodeName`
- `definitionNodeType`

并通过字段映射绑定数据库列：

- `definition_node_id`
- `definition_node_name`
- `definition_node_type`

### 3. 运行层使用口径

运行层代码中凡是读取 `tb_workflow_node_instance` 的这三个字段，都按“节点定义快照”理解：

- 节点实例创建时，从 `tb_workflow_node` 写入
- 审批记录中写入的节点实例名称、类型，也基于该定义快照
- 对外返回当前运行节点时，仍可继续返回 `nodeId/nodeName/nodeType`，但其语义是“当前运行节点对应的节点定义信息”
