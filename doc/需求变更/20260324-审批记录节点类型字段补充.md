# 审批记录补充节点实例冗余字段

## 变更原因

当前 `tb_workflow_approval_record` 只记录审批动作、操作人和节点实例主键，缺少操作发生时对应节点实例、来源节点、目标节点的类型和名称冗余信息。
后续在审批轨迹展示、运行分析和异常排查时，需要直接看到该记录对应的是哪个节点、属于什么节点类型，避免每次再通过节点实例和节点定义做额外关联解析。

## 变更内容

- `tb_workflow_approval_record` 新增字段 `node_instance_type`
- `tb_workflow_approval_record` 新增字段 `node_instance_name`
- `tb_workflow_approval_record` 新增字段 `from_node_type`
- `tb_workflow_approval_record` 新增字段 `from_node_name`
- `tb_workflow_approval_record` 新增字段 `to_node_type`
- `tb_workflow_approval_record` 新增字段 `to_node_name`
- `node_instance_type` / `node_instance_name`：记录审批动作实际发生所在节点实例的类型与名称
- `from_node_type` / `from_node_name`：记录本次流转来源节点实例的类型与名称
- `to_node_type` / `to_node_name`：记录本次流转目标节点实例的类型与名称
- 取值口径与 `tb_workflow_node.node_type`、`tb_workflow_node_instance.node_type` 保持一致：
  - `START`
  - `APPROVAL`
  - `CONDITION`
  - `PARALLEL_SPLIT`
  - `PARALLEL_JOIN`
  - `END`

## 影响说明

- 影响 SQL：`tb_workflow_approval_record`
- 影响代码：
  - `WorkflowApprovalRecord` 实体
  - 运行层后续写审批记录时，需要同步写入 3 组节点实例冗余字段
- 本次不新增 controller
