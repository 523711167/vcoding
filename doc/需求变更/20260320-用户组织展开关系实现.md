# 用户组织展开关系实现

## 变更原因

当前 `tb_user_dept_rel` 同时承载“用户直接绑定组织”和“用户在组织树上的生效覆盖范围”两种语义，会导致后续查询“某组织下有哪些用户”时无法区分直接绑定与祖先继承结果，也无法稳定追溯来源关系。

## 变更内容

- 新增 `tb_user_dept_rel_expand`，专门存放用户组织绑定向下展开后的当前生效节点。
- 保留 `tb_user_dept_rel` 作为直接绑定事实表，不再复用为展开结果表。
- 用户执行 `/sys/user/depts/update` 后，同一事务内全量重建该用户的展开关系。
- 组织新增、删除、移动、状态调整、岗位类型调整后，仅重建受影响用户的展开关系。
- 查询“某组织下用户”时改为读取 `tb_user_dept_rel_expand`，并按 `user_id` 去重。

## 影响说明

- 数据库：新增迁移 SQL `sql/20260320.sql`，创建并回填 `tb_user_dept_rel_expand`。
- 后端代码：新增展开关系实体、Mapper、重建服务，并调整用户组织分配与组织管理逻辑。
- 测试：补充用户组织展开重建和按组织查用户去重相关测试。
