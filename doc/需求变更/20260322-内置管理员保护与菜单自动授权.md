# 内置管理员保护与菜单自动授权

## 变更原因

当前菜单新增后只写入 `tb_sys_menu`，不会自动建立角色菜单关系。  
这会导致新菜单创建完成后，任何用户都暂时不可见，连系统管理员也需要额外执行一次角色授权操作才能看到。

同时，系统默认初始化了内置管理员账号 `admin` 和内置超级管理员角色 `ADMIN`。  
这两类内置数据属于系统保底能力，不应在日常管理操作中被误修改、误停用或误删除，否则会直接影响系统维护入口与权限基线。

## 变更内容

### 1. 菜单新增自动授权给 `ADMIN`

- 菜单创建成功后，系统自动为角色编码 `ADMIN` 新增一条角色菜单关联
- 若 `ADMIN` 角色不存在，则本次菜单创建直接失败并返回明确业务异常
- 本次自动授权只补当前新建菜单本身，不额外重建全量角色菜单关系

### 2. 内置管理员账号 `admin` 禁止修改

以下接口涉及到目标用户为 `admin` 时，统一返回业务异常，不允许继续执行：

- 用户修改
- 用户删除
- 用户重置密码
- 用户状态修改
- 用户角色分配
- 用户组织分配

查询类接口不受影响。

### 3. 内置超级管理员角色 `ADMIN` 禁止修改

以下接口涉及到目标角色编码为 `ADMIN` 时，统一返回业务异常，不允许继续执行：

- 角色修改
- 角色删除
- 角色状态修改
- 角色菜单分配
- 角色数据权限修改

查询类接口不受影响。

## 影响说明

- 影响模块：
  - 菜单管理
  - 用户管理
  - 角色管理
- 影响代码：
  - `MenuServiceImpl`
  - `UserServiceImpl`
  - `RoleServiceImpl`
- 本次不涉及表结构调整，不需要新增 SQL
- 需要补充服务层测试，覆盖：
  - 新增菜单自动授权 `ADMIN`
  - `admin` 用户写操作保护
  - `ADMIN` 角色写操作保护
