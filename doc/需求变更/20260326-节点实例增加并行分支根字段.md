# 节点实例增加并行分支根字段

## 变更原因

在审批运行过程中，仅依赖审批记录表逐级回溯，才能判断当前节点是否处于某个并行拆分分支中。  
当并行拆分后的某条分支内存在多个审核节点、条件节点甚至嵌套网关时，这种回溯方式复杂且不利于后续拒绝、聚合、分支状态判断。

因此本次在 `tb_workflow_node_instance` 中新增并行分支根节点冗余字段，直接标识当前节点所属分支来自哪个拆分节点实例。

## 变更内容

### 1. 表结构调整

`tb_workflow_node_instance` 新增：

- `parallel_branch_root_id`：所属并行分支根拆分节点实例 ID

字段口径：

- 仅表示当前节点所属并行分支的“最近一层”拆分根节点实例 ID
- 不存节点定义 ID，存节点实例 ID
- 非并行分支节点取 `NULL`
- `PARALLEL_JOIN` 节点取 `NULL`

### 2. 运行时写入规则

节点实例创建时：

- 如果来源节点实例本身是 `PARALLEL_SPLIT`，则当前节点的 `parallel_branch_root_id = 来源拆分节点实例ID`
- 如果来源节点实例已存在 `parallel_branch_root_id`，则当前节点继续继承该值
- 如果目标节点是 `PARALLEL_JOIN`，则当前节点不继承，固定为 `NULL`
- 若发生嵌套并行拆分，则子并行分支节点以“最近一层拆分节点实例”作为新的根

### 3. 使用口径

后续若需要判断某个节点是否处于并行分支内，可直接判断：

- `parallel_branch_root_id IS NOT NULL`

若需要判断两个节点是否属于同一并行分支，可比较：

- `parallel_branch_root_id`
