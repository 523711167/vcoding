# 流程定义移除业务编码字段

## 变更原因

当前 `tb_workflow_definition` 中保留了 `biz_code` 字段，但业务定义与流程定义的绑定关系已经由业务定义表自身维护，流程定义表不再需要冗余保存业务编码。

继续保留该字段会造成两层绑定口径并存：

- 一层是业务定义表维护的正式绑定关系；
- 一层是流程定义表上的冗余业务编码字段。

这会增加数据一致性维护成本，也会让流程定义查询和接口契约出现不必要的字段。

## 变更内容

- 从 `tb_workflow_definition` 中删除 `biz_code` 字段。
- 工作流定义相关接口移除 `bizCode` 入参、出参和查询条件。
- 工作流定义对象映射、持久化实体、查询对象、返回对象同步移除 `bizCode`。
- 工作流定义列表与分页查询不再支持按 `bizCode` 检索。

## 影响说明

- 数据库影响：
  - 新增迁移 SQL，执行 `ALTER TABLE tb_workflow_definition DROP COLUMN biz_code`。
- 接口影响：
  - `/sys/workflow-definition/create`
  - `/sys/workflow-definition/update`
  - `/sys/workflow-definition/list`
  - `/sys/workflow-definition/page`
  - `/sys/workflow-definition/detail`
  以上接口返回或入参中不再包含 `bizCode`。
- 代码影响：
  - 工作流定义实体、ETO、QTO、VO、MapStruct 映射、Service 查询条件同步调整。
- 兼容性说明：
  - 旧调用方若继续传 `bizCode`，后端 DTO 不再接收该字段，等价于静默忽略。
