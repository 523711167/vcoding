# 流程实例增加业务标识字段

## 变更原因

当前 `tb_workflow_instance` 只保存流程定义信息和申请人信息，缺少与业务定义的直接关联字段。  
这会导致流程实例查询、业务维度统计以及后续按业务做运行态过滤时，仍需要回查 `tb_biz_apply` 或流程定义层才能定位所属业务。

因此本次为 `tb_workflow_instance` 新增 `biz_id` 字段，直接冗余业务定义主键。

## 变更内容

### 1. 表结构调整

`tb_workflow_instance` 新增：

- `biz_id`：业务定义 ID

字段口径：

- `biz_id` 对应 `tb_biz_definition.id`
- 在流程实例创建时一次性写入
- 历史实例通过业务申请表回填

### 2. 流程发起链路调整

`WorkflowLaunchServiceImpl.startWorkflow(...)` 创建流程实例时：

- 从业务申请的 `bizDefinitionId` 取值
- 同步写入 `tb_workflow_instance.biz_id`

### 3. 使用收益

- 查询流程实例时可直接按业务维度过滤
- 后续待办、已办、统计分析可直接基于流程实例表按业务定义聚合
- 降低运行层查询对业务申请表的依赖
