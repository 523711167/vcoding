# 节点审批人实例字段调整

## 变更原因

当前 `tb_workflow_node_approver_instance` 使用 `handled_at` 表达审批人处理时间，字段语义不够统一。  
同一运行层中，流程实例与节点实例都统一使用 `finished_at` 表达完成时间，因此审批人实例也需要统一口径。

另外，`delegate_to` 目前只保存转交目标用户 ID，待办、已办、审批轨迹展示时仍需要额外回查用户表，因此补充转交目标用户名冗余字段。

## 变更内容

### 1. 表结构调整

`tb_workflow_node_approver_instance` 调整为：

- `handled_at` 重命名为 `finished_at`
- 新增 `delegate_to_name`

字段口径：

- `finished_at`：审批人实例完成时间，适用于通过、拒绝、转交、取消等结束态
- `delegate_to_name`：转交目标用户姓名冗余，仅在转交场景有值

### 2. 代码调整

- `WorkflowNodeApproverInstance` 实体字段同步改为：
  - `finishedAt`
  - `delegateToName`

### 3. 历史数据处理

- 保留原 `handled_at` 历史值，通过字段重命名迁移到 `finished_at`
- 根据 `delegate_to` 回填 `delegate_to_name`
