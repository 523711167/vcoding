# 业务申请冗余业务名称

## 变更原因

当前业务申请草稿列表与详情在展示业务名称时，需要额外通过 `bizDefinitionId` 关联查询业务定义表。为降低后续查询耦合和展示成本，需要在业务申请表冗余保存业务名称快照。

## 变更内容

1. 在 `tb_biz_apply` 新增 `biz_name` 字段，作为业务定义名称冗余快照。
2. 新增数据库迁移 SQL：
   - 为历史库按需新增 `biz_name` 字段（幂等）。
   - 通过 `tb_biz_definition.biz_name` 回填历史业务申请数据。
3. 在业务申请草稿保存、草稿修改场景，按当前 `bizDefinitionId` 对应业务定义同步写入 `biz_name`。
4. 更新 `init.sql` 中 `tb_biz_apply` DDL，保持初始化脚本与当前结构一致。

## 影响说明

- 影响模块：
  - 业务申请服务 `BizApplyServiceImpl`
  - 业务申请持久化实体 `BizApply`
- 影响数据库：
  - `sql/20260401.sql`
  - `sql/init.sql`
- 接口契约：
  - 本次不新增入参，不修改现有接口 URL，不改变响应结构。
