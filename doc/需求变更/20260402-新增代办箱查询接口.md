# 新增代办箱查询接口

## 变更原因

前端需要在“代办箱”页面展示当前登录用户待处理审批任务，并支持列表与分页两种查询能力，便于在工作台快速办理审批。

## 变更内容

1. 新增工作流代办箱查询接口：
   - `GET /sys/biz-apply/todo/list`
   - `GET /sys/biz-apply/todo/page`
2. 新增代办箱查询参数对象：
   - `WorkflowTodoListQTO`
   - `WorkflowTodoPageQTO`
3. 新增代办箱返回对象：
   - `WorkflowTodoVO`
4. 新增查询服务：
   - `WorkflowBizQueryService`
   - `WorkflowBizQueryServiceImpl`
5. 查询口径统一：
   - 仅返回当前登录用户、状态为 `PENDING`、且 `isActive=YES` 的审批人实例。
   - `list` 与 `page` 复用同一筛选逻辑，支持按 `bizApplyId`、`bizDefinitionId`、`title` 条件过滤。
6. 查询实现调整为关联查询：
   - 以 `tb_workflow_node_approver_instance` 作为主表。
   - 关联 `tb_workflow_instance`、`tb_biz_apply` 一次性返回代办展示字段。
7. 新增单元测试覆盖代办箱查询核心行为。

## 影响说明

- 影响接口：
  - `WorkflowBizController`
- 新增对象：
  - `qto/workflow/WorkflowTodoListQTO`
  - `qto/workflow/WorkflowTodoPageQTO`
  - `vo/workflow/WorkflowTodoVO`
- 新增服务：
  - `WorkflowBizQueryService`
  - `WorkflowBizQueryServiceImpl`
- 测试影响：
  - 新增 `WorkflowBizQueryServiceImplTests`
- 数据库结构：
  - 本次不涉及 SQL 变更。
