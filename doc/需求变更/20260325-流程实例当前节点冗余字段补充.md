# 流程实例当前节点冗余字段补充

## 变更原因

当前 `tb_workflow_instance` 只有 `current_node_id`，缺少当前节点名称和类型冗余字段。  
这会导致流程实例列表、详情、待办查询在展示当前节点时，仍需要额外回查节点定义层或节点实例表。

因此本次补充当前节点快照字段，统一沉淀到流程实例表。

## 变更内容

### 1. 表结构调整

`tb_workflow_instance` 新增：

- `current_node_name`：当前所在节点名称
- `current_node_type`：当前所在节点类型

字段口径：

- 与 `current_node_id` 保持同一语义，表示流程实例当前主运行位置的节点定义快照
- 流程仍在运行中时，三个字段同步回写
- 流程已经结束时，三个字段统一置空
- 历史数据通过 `tb_workflow_node` 按 `current_node_id` 回填

### 2. 流程发起链路调整

`WorkflowLaunchServiceImpl.startWorkflow(...)` 在首批节点激活完成后：

- 同步写入 `current_node_id`
- 同步写入 `current_node_name`
- 同步写入 `current_node_type`

补充规则：

- 串行审批场景下，指向当前审批节点
- 并行拆分场景下，仍按既有口径指向当前主网关位置
- 如果提交流程后直接到结束节点，则三个字段都不保留
