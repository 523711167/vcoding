# 工作流节点审批人值改为 bigint

## 变更原因

当前 `tb_workflow_node_approver.approver_value` 使用字符串字段保存用户、角色、组织主键，数据库层和 Java 层都存在重复转换，角色审批人与组织审批人的关联查询还需要显式 `CAST`。  
该字段实际语义已经收敛为“单条记录只保存一个主体 ID”，继续使用字符串没有必要。

## 变更内容

- 将 `tb_workflow_node_approver.approver_value` 从字符串字段调整为 `BIGINT`
- Java 实体 `WorkflowNodeApprover.approverValue` 调整为 `Long`
- 流程定义保存入参 `WorkflowNodeApproverETO.approverValue` 调整为 `Long`
- 流程定义详情出参 `WorkflowNodeApproverVO.approverValue` 调整为 `Long`
- 流程 JSON 解析时，前端传入的数字或数字字符串统一转为 `Long`
- 保留“一条记录一个审批主体”的规则，若前端仍传逗号拼接值，继续按非法配置处理
- 角色审批人与组织审批人的解析逻辑改为按数值主键直接匹配，不再依赖字符串比较

## 影响说明

- 影响表：`tb_workflow_node_approver`
- 影响模块：流程定义保存、流程定义详情、工作流发起时审批人解析、组织审批人展开关系重建
- 影响 SQL：新增迁移文件 [20260327.sql](/Users/xixipeng/Downloads/vCoding/sql/20260327.sql)
- 影响测试：流程定义服务测试、流程发起测试、审批组织展开测试、运行层基础服务测试需同步更新类型口径
