# 需求变更文档目录

本目录用于承载已形成设计文档之后的新增需求、需求调整和设计变更记录。

## 维护规则

- `doc/vCoding后台管理系统设计/` 下的原始设计文档默认不再直接修改。
- 若有需求变更，统一在本目录新增独立 Markdown 文档记录，不直接覆盖原设计文档。
- 仅在自动化任务统一执行“合并需求设计”时，才将本目录中的变更内容合并回原设计文档。

## 文档要求

- 文件名建议使用 `yyyyMMdd-变更主题.md`。
- 文档中必须明确说明“为什么要进行此次需求变更”。
- 文档中必须明确说明“此次需求变更的具体内容”。

## 建议模板

```md
# <需求变更标题>

## 变更原因

说明此次为什么要进行需求变更。

## 变更内容

说明本次需求变更涉及的功能、范围、规则、接口、数据或交互调整。

## 影响说明

说明影响的模块、设计文档、接口、数据库或联调范围。
```
