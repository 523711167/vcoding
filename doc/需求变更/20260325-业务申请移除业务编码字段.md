# 业务申请移除业务编码字段

## 变更原因

当前 `tb_biz_apply` 通过 `biz_code` 关联业务定义，业务申请表对业务定义仍然依赖编码字段。  
随着系统内多个模块逐步收敛为主键关联模式，继续在运行层表中保存 `biz_code` 会带来两类问题：

- 运行层仍依赖业务编码做关联，不利于统一主键关联口径
- 后续若需要做业务定义重命名、编码规则收敛或编码冻结，运行层会继续承担额外耦合

因此本次将 `tb_biz_apply` 从“保存业务编码”调整为“直接保存业务定义主键”。

## 变更内容

### 1. 表结构调整

`tb_biz_apply`：

- 移除字段 `biz_code`
- 新增字段 `biz_definition_id`

语义调整为：

- `biz_definition_id` 直接关联 `tb_biz_definition.id`
- 业务申请提交审批时，不再根据 `biz_code` 查业务定义，而是直接根据 `biz_definition_id` 查业务定义

### 2. 流程发起链路调整

`WorkflowLaunchServiceImpl.startWorkflow(...)` 调整为：

- 从 `BizApply.bizDefinitionId` 直接查询业务定义
- 校验业务定义启用状态
- 基于该业务定义继续校验发起权限和流程定义状态

### 3. 历史数据迁移

迁移 SQL 需要：

- 先新增 `biz_definition_id`
- 按原 `biz_code` 与 `tb_biz_definition.biz_code` 回填 `biz_definition_id`
- 在确认历史数据回填完成后，删除 `biz_code`
