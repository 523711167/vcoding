# 流程节点超时字段改为分钟

## 变更原因

当前工作流节点中的超时和提醒字段按“小时”命名与存储，但前端流程设计器实际传递的是分钟值。

继续保留小时口径会造成前后端语义错位，轻则展示不一致，重则导致超时计算结果错误。

## 变更内容

- 工作流节点超时字段由小时口径改为分钟口径。
- `tb_workflow_node.timeout_hours` 调整为 `timeout_minutes`。
- `tb_workflow_node.remind_hours` 调整为 `remind_minutes`。
- Java 实体、VO、内部节点 DTO 同步改为：
  - `timeoutMinutes`
  - `remindMinutes`
- 前端传入的 `timeoutAfterMinutes`、`remindAfterMinutes` 不再做小时换算，按分钟原值保存。

## 影响说明

- 数据库影响：
  - 更新迁移 SQL `sql/20260323.sql`，将旧列重命名为分钟口径列名。
- 接口影响：
  - 流程定义详情中的节点字段改为分钟语义。
- 代码影响：
  - 工作流定义保存逻辑不再执行分钟转小时的换算。
  - 后续流程实例计算超时截止时间时，也应基于分钟口径处理。
