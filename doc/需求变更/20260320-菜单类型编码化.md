# 菜单类型编码化

## 变更背景

`tb_sys_menu.type` 原先使用 `1/2/3` 存储目录、菜单、按钮类型。  
当前系统中的组织类型、数据权限类型等枚举字段已经统一使用字符串 `code` 存储，菜单类型继续使用整型会导致：

- 数据模型口径不统一
- 接口入参和出参语义不直观
- 前后端联调时需要额外维护数字和枚举名称的映射

因此本次将菜单类型统一调整为 `MenuTypeEnum.code`。

## 变更内容

### 数据库

- `tb_sys_menu.type` 从 `TINYINT` 调整为 `VARCHAR(16)`
- 存储值统一改为：
  - `DIRECTORY`
  - `MENU`
  - `BUTTON`
- 历史数据 `1/2/3` 通过迁移 SQL 转换为对应 code

### 后端代码

- `SysMenu.type` 调整为 `String`
- 菜单新增、编辑、树查询接口中的 `type` 参数改为字符串编码
- 菜单服务层所有类型判断统一改为基于 `MenuTypeEnum.code`
- 菜单详情与菜单树的 `typeMsg` 改为根据 `code` 解析
- 角色菜单授权补齐逻辑改为按 `MENU` 类型判断菜单节点

### 初始化 SQL

- 基线 SQL 中 `tb_sys_menu.type` 改为字符串字段
- 初始化菜单数据直接写入 `DIRECTORY/MENU/BUTTON`

## 迁移口径

执行 [20260321.sql](/Users/xixipeng/Downloads/vCoding/sql/20260321.sql) 后：

1. 修改 `tb_sys_menu.type` 字段类型
2. 将历史值 `1/2/3` 转为 `DIRECTORY/MENU/BUTTON`
3. 新增数据按 code 直接写入，不再使用数字

## 兼容说明

- 本次变更后，菜单相关接口不再接受 `1/2/3`
- 调用方需要改为传入：
  - `DIRECTORY`
  - `MENU`
  - `BUTTON`
