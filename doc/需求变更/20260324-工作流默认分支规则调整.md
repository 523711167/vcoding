# 工作流默认分支规则调整

## 变更原因

现有流程连线设计仅通过 `condition_expr = NULL` 隐式表达无条件或兜底路径，无法稳定区分“普通无条件连线”和“默认分支”。
在 `CONDITION` 与 `PARALLEL_SPLIT` 节点存在多条出边时，如果缺少显式默认分支，运行时可能出现所有条件都不命中导致流程卡死的问题。

## 变更内容

### 1. 连线表字段调整

- `tb_workflow_transition` 新增 `is_default`
- 口径统一为：
  - `condition_expr`：表示条件表达式；`NULL` 仅表示无条件
  - `is_default=1`：表示该连线是默认分支

### 2. 流程配置规则调整

- `CONDITION` 节点在存在多条出边时，必须且只能配置一条默认分支
- `PARALLEL_SPLIT` 节点在存在多条出边时，必须且只能配置一条默认分支
- `CONDITION` 节点运行时按 `priority` 依次判断非默认分支，只取第一条命中的分支；若都未命中，则走默认分支
- `PARALLEL_SPLIT` 节点运行时遍历全部非默认分支，激活所有命中的分支；若都未命中，则走默认分支

### 3. 前端 JSON 映射规则

- 前端边属性中的 `properties.is_default` 映射到 `tb_workflow_transition.is_default`
- 前端边属性中的 `properties.expression` 映射到 `condition_expr`
- 前端边属性中的 `properties.priority` 映射到 `priority`

### 4. 后端校验要求

- 创建流程定义时校验默认分支规则
- 修改流程定义时校验默认分支规则
- 发布流程定义时再次校验默认分支规则
- 不满足规则时返回标准业务异常，不允许保存或发布
