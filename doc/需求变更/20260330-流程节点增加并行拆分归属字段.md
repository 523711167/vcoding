# 流程节点增加并行拆分归属字段

## 变更原因

当前 `tb_workflow_node` 只保存节点本身的定义信息，无法直接判断某个定义节点是否处于某个并行拆分段内。  
在并行分支存在多个审批节点、条件节点、嵌套网关时，仅依赖连线关系逐级回溯所属并行拆分节点，后续运行时处理和规则判断都不够直接。

## 变更内容

- 在 `tb_workflow_node` 新增字段 `parallel_split_node_id`
- 字段语义收敛为：当前节点所属的最近一层并行拆分节点定义 ID
- 字段写入规则：
  - 普通不在并行段内的节点，`parallel_split_node_id = NULL`
  - 并行拆分分支内的节点，写所属 `PARALLEL_SPLIT` 节点定义 ID
  - `PARALLEL_JOIN` 节点也写所属 `PARALLEL_SPLIT` 节点定义 ID
  - 嵌套并行场景下，取最近一层并行拆分节点定义 ID
- `PARALLEL_SPLIT` 节点本身不写自己，顶层拆分节点默认 `NULL`
- 该字段不要求前端传入，由后端在保存流程定义时根据节点与连线结构自动计算并落库

## 影响说明

- 影响数据库表：`tb_workflow_node`
- 影响代码模块：
  - `WorkflowNode` 实体
  - `WorkflowDefinitionServiceImpl` 流程定义保存逻辑
  - `WorkflowDefinitionStructMapper` 节点对象转换
  - `WorkflowNodeVO` 节点详情返回
- 影响 SQL：新增 `sql/20260330.sql`
- 影响测试：补充 `WorkflowDefinitionServiceImplTests`，验证并行拆分、并行聚合节点的归属字段写入结果
