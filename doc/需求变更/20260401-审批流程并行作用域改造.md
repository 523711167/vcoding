# 审批流程并行作用域改造

## 变更原因

当前审批流程运行层已支持并行拆分、并行聚合和节点实例冗余字段，但在“并行拆分中嵌套并行拆分”的场景下，仅依赖 `tb_workflow_node.parallel_split_node_id` 和 `tb_workflow_node_instance.parallel_branch_root_id` 已无法完整表达运行时层级关系。

主要问题包括：

- 无法准确区分“同一流程实例中某一层具体并行段”的运行时归属
- 并行聚合节点难以稳定判断当前应汇聚的分支范围
- 内层并行聚合完成后，后续节点回到父层并行段的语义不够清晰
- 后续若继续扩展回路、重入或更复杂的并行流转，现有字段支撑能力不足

因此本次需求变更引入“并行作用域”模型，将运行时并行层级从“节点推导”收敛为“显式数据建模”。

## 变更内容

### 1. 定义层字段口径保持不变

继续保留：

- `tb_workflow_node.parallel_split_node_id`

字段语义不变：

- 表示当前定义节点所属的最近一层并行拆分节点定义 ID
- 用于表达流程图定义层的结构归属
- 供运行时创建并行作用域、定位父子层级时辅助使用

本次不新增 `parallel_owner_split_node_id`。

### 2. 运行层新增并行作用域表

新增表：

- `tb_workflow_parallel_scope`

表职责：

- 表达某一次流程实例运行过程中实际产生的一层并行段
- 明确记录该并行段对应的拆分节点、聚合节点、父作用域、到齐计数和最终状态

本次新增的主要字段包括：

- `instance_id`：流程实例 ID
- `definition_id`：流程定义 ID
- `split_definition_node_id/name/type`：并行拆分节点定义快照
- `join_definition_node_id/name/type`：并行聚合节点定义快照
- `parent_scope_id`：父并行作用域 ID，最外层为空
- `status`：并行作用域状态
- `expected_branch_count`：理论应汇聚分支数
- `arrived_branch_count`：当前已到达聚合节点的分支数
- `finished_at`：作用域完成时间

### 3. 节点实例新增并行作用域字段

在 `tb_workflow_node_instance` 中新增：

- `parallel_scope_id`

字段语义：

- 表示当前节点实例所属的运行时并行作用域 ID
- 以后节点实例所属并行层级、并行聚合到齐判断、嵌套并行回到父层等逻辑，统一基于该字段处理

### 4. 现有 `parallel_branch_root_id` 进入兼容保留阶段

`tb_workflow_node_instance.parallel_branch_root_id` 暂时保留，不立即删除。

当前口径调整为：

- 新增的运行时主逻辑逐步迁移到 `parallel_scope_id`
- `parallel_branch_root_id` 仅作兼容保留，不再作为后续嵌套并行主判断字段
- 待后续业务逻辑全部切换完成后，再评估是否删除该字段

### 5. 本次代码落地范围

本轮先完成基础设施落地，不直接改审核流转逻辑。

已落地内容包括：

- 新增迁移 SQL：`sql/20260401.sql`
- 同步初始化脚本：`sql/init.sql`
- 新增实体：`WorkflowParallelScope`
- 新增 Mapper：`WorkflowParallelScopeMapper`
- 新增 Service：`WorkflowParallelScopeService`
- 新增 ServiceImpl：`WorkflowParallelScopeServiceImpl`
- 更新 `WorkflowNodeInstance`，增加 `parallelScopeId`

审核通过、审核拒绝、并行拆分、并行聚合、嵌套并行回父层等业务逻辑，将在后续实现中基于该结构继续落地。

## 影响说明

本次需求变更影响范围如下：

- 数据库
  - 新增 `tb_workflow_parallel_scope`
  - `tb_workflow_node_instance` 新增 `parallel_scope_id`
- 初始化脚本
  - `sql/init.sql` 需要同步包含上述结构
- Java 代码
  - 新增并行作用域相关实体、Mapper、Service、ServiceImpl
  - 更新节点实例实体字段
- 审批流程运行层设计
  - 后续并行拆分/聚合和嵌套并行逻辑，统一以 `parallel_scope_id` 为核心运行时归属字段

本次变更暂不直接修改接口契约，不新增 Controller，也不调整现有审核 API 入参与出参。
