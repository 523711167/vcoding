# 流程连线节点冗余字段调整

## 变更原因

当前 `tb_workflow_transition` 只保存 `from_node_id` 和 `to_node_id`，连线本身缺少来源节点、目标节点的名称与类型冗余字段。  
这会带来两个问题：

- 查询流程定义连线详情时，若只看连线数据，无法直接展示节点名称和节点类型
- 节点定义后续发生名称调整时，连线侧缺少独立快照，不利于定义层排查和历史分析

因此需要在流程连线表中补充节点名称、节点类型冗余字段，并在保存流程定义时由后端统一写入。

## 变更内容

### 1. 表结构调整

`tb_workflow_transition` 新增 4 个字段：

- `from_node_name`：来源节点名称冗余
- `from_node_type`：来源节点类型冗余
- `to_node_name`：目标节点名称冗余
- `to_node_type`：目标节点类型冗余

说明：

- 这些字段由流程定义保存时根据节点配置自动写入
- 前端仍只传节点编码，不需要新增传参

### 2. 保存口径调整

在流程定义新增、修改、版本复制后的子表持久化阶段：

- 先根据节点参数建立 `code -> 节点快照` 映射
- 保存连线时，除写入 `from_node_id / to_node_id` 外，同时写入对应的名称和类型冗余值

### 3. 查询口径调整

流程定义详情返回的 `transitionList` 增加以下字段：

- `fromNodeName`
- `fromNodeType`
- `toNodeName`
- `toNodeType`

## 兼容说明

- 旧前端参数不需要调整
- 已有历史数据通过 SQL 按节点表回填冗余字段
- 后续节点名称变更时，新保存的连线写入最新冗余值；历史旧版本数据不回写
