# 流程定义更新版本发布规则

## 变更原因

当前流程定义修改接口只允许编辑草稿版本，已发布和已停用版本会直接报“仅草稿流程允许修改”。

这与实际业务口径不一致。流程一旦进入发布或停用状态，修改时应生成新版本并直接发布，旧版本自动转为停用，不能要求调用方先手工复制草稿再发布。

## 变更内容

- `草稿` 状态：
  - 继续按原口径直接编辑当前记录。
  - 不新建版本。
- `已发布` 状态：
  - 修改时新建一条新版本记录。
  - `version = 当前最大版本号 + 1`
  - 新版本直接发布。
  - 同流程编码下旧版本自动停用。
- `已停用` 状态：
  - 修改时与“已发布状态”保持同一处理逻辑。
  - 新建新版本并直接发布。
  - 同流程编码下旧版本保持停用，若存在其他已发布旧版本也自动停用。
- `tb_workflow_node_approver_dept_expand`：
  - 不复制历史展开结果。
  - 按新版本写入后的最新审批人 direct 配置重新全量重建。
- `tb_workflow_node`、`tb_workflow_node_approver`、`tb_workflow_transition`：
  - 按本次最新 `workFlowJson` 重新解析并写入新版本数据。

## 影响说明

- 数据库影响：
  - 无新增表结构变更。
- 接口影响：
  - `/sys/workflow-definition/update` 的行为由“仅可编辑草稿”调整为“按状态自动选择直接编辑或新建发布版本”。
- 代码影响：
  - 工作流定义服务层需要在更新时区分 `DRAFT/PUBLISHED/DISABLED` 三种状态处理。
  - 需要补充版本递增、旧版本停用、新版本发布的测试覆盖。
