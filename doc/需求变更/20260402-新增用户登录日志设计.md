# 新增“用户登录日志”设计

## 1. 变更背景

当前系统仅在登录成功后更新 `tb_user.last_login_at`，缺少完整登录留痕能力。  
为满足审计与问题排查，需要新增登录日志记录能力。

## 2. 业务口径冻结（本次实现）

### 2.1 记录范围

- 仅记录 `/oauth2/token` 且 `grant_type=password_login` 的登录尝试。
- 包含“登录成功”和“登录失败”两类事件。

### 2.2 记录内容

- 成功日志：`userId`、`username`、`result=SUCCESS`、`loginAt`、`clientIp`、`userAgent`。
- 失败日志：`username`（若可解析）、`result=FAIL`、`failReason`、`loginAt`、`clientIp`、`userAgent`。
- 失败场景 `userId` 允许为空。

### 2.3 非目标范围

- 本次不新增登录日志查询 API（仅服务端留痕）。
- 不改变现有 `/oauth2/token` 成功/失败返回契约。

## 3. 实现蓝图

### 3.1 数据层

- 新增表：`tb_login_log`。
- 字段：`id/user_id/username/result/fail_reason/client_ip/user_agent/login_at/created_at/updated_at/is_deleted`。
- 索引：`user_id`、`username`、`result`、`login_at`。
- SQL 同步：
  - 增量脚本：`sql/20260402.sql`
  - 全量基线：`sql/init.sql`

### 3.2 代码分层

- 新增实体：`LoginLog`
- 新增 Mapper：`LoginLogMapper`
- 新增 Service：`LoginLogService`
- 新增 Struct：`LoginLogStructMapper`
- 新增枚举：`LoginResultEnum`
- 新增 Controller 查询接口（认证模块）：
  - `GET /sys/auth/login-log/list`
  - `GET /sys/auth/login-log/page`
  - `GET /sys/auth/login-log/detail`

### 3.3 认证链路接入

- 登录成功：
  - 继续更新 `tb_user.last_login_at`。
  - 在 `AuthenticationSuccessEventListener` 调用 `LoginLogService.recordSuccess(...)` 写成功日志。
- 登录失败：
  - 在 `OAuth2TokenEndpointFailureHandler` 调用 `LoginLogService.recordFailure(...)` 写失败日志。
  - 仅对 `password_login` 分支记录失败日志。

### 3.4 客户端信息口径

- `clientIp` 优先读取请求头：`X-Forwarded-For`、`X-Real-IP`，回退 `remoteAddr`。
- `userAgent` 读取请求头 `User-Agent`。

## 4. 验收口径

1. 用户名不存在或密码错误时，返回原有错误结构，同时新增一条 `FAIL` 日志。
2. 登录成功后，`tb_user.last_login_at` 被更新，同时新增一条 `SUCCESS` 日志。
3. 非 `password_login` 的 token 失败不写登录失败日志。
